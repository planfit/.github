## 필수 검증 규칙 (반드시 지적할 것)

### 🟡 타입 안전성
- `any` 타입 사용 금지 — 구체적 타입 또는 `unknown` 사용

### 🟡 로깅 규칙
- `console.*` 직접 사용 금지 — 프로젝트 로깅 유틸리티 사용

## 제외 규칙 (지적하지 말 것)

- import 순서 오류 (자동 포매터가 처리)
- 서버 API 설계 변경 제안 (클라이언트만으로 서버 의도를 추측하지 말 것)

## 자기 검증 (코멘트 작성 전 반드시 수행)

- "이 지적이 틀릴 수 있는 이유"를 먼저 생각하라
- 아키텍처/데이터 구조 변경 제안: 서버 API를 모르므로 지적하지 마라
- 동시성/race condition: 기존 보호 메커니즘을 확인하고, 구체적 우회 시나리오를 설명할 수 없으면 지적하지 마라
- 확신이 낮은 이슈는 제외하라

## 경로별 리뷰 포커스 (레포에 맞게 수정)

| 경로 | 포커스 |
|------|--------|
| `src/**/*.{ts,tsx}` | 코드 컨벤션, 베스트 프랙티스 |
| `tests/**` | 테스트 커버리지, 엣지 케이스 |
